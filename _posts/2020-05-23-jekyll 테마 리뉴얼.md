## 하단, 업데이트 날짜 부분 날리기 🪂

<1>

내 블로그 첫 화면이다. 왼쪽에 커다랗게 있는 프로필을 포스트 하단으로 위치시키기 전에 `태그`, `카테고리`를 상단으로 옮기고, 업데이트 날짜 부분은 이미 상단에 한번 표기되 있으므로 없앨 것이다.

`_layouts/single.html`에 64~68 줄에 보면 

```html
{% raw %}{% include page__taxonomy.html %}
{% if page.last_modified_at %}
	<p class="page__date"><strong><i class="fas fa-fw fa-calendar-alt" aria-hidden="true"></i> {{ site.data.ui-text[site.locale].date_label | default: "Updated:" }}</strong> <time datetime="{{ page.last_modified_at | date: "%Y-%m-%d" }}">{{ page.last_modified_at | date: "%B %-d, %Y" }}</time></p>
{% elsif page.date %}
	<p class="page__date"><strong><i class="fas fa-fw fa-calendar-alt" aria-hidden="true"></i> {{ site.data.ui-text[site.locale].date_label | default: "Updated:" }}</strong> <time datetime="{{ page.date | date_to_xmlschema }}">{{ page.date | date: "%B %-d, %Y" }}</time></p>
{% endif %}{% endraw %}
```

이렇게 되어있는데 `{% raw %}{% include page__taxonomy.html %}{% endraw %}`이 부분이 태그와 카테고리가 선언되어있는 html을 include 한 것이고, 그 밑에 부분이 업데이트 날짜부분을 표기한 부분이다. `{% raw %}{% include page__taxonomy.html %}{% endraw %}` 부분만 빼고 날짜 부분을 전부 날려주자. 그러면 하단에 날짜부분이 전부 날라갔다.☺️



## 태그, 카테고리를 상단으로 옮기기 🏋️‍♀️

두번째로, 태그와 카테고리 부분을 상단으로 옮길 것인데 아까 `{% raw %}{% include page__taxonomy.html %}{% endraw %}`이 부분을 그대로 `<head>`제일 하단에 위치시킨다. 

```html
{% raw %}<header>
	{% if page.title %}<h1 id="page-title" class="page__title" itemprop="headline">{{ page.title | markdownify | remove: "<p>" | remove: "</p>" }}</h1>{% endif %}
	{% if page.read_time %}
          <!--
            <p class="page__meta"><i class="far fa-clock" aria-hidden="true"></i> {% include read-time.html %}</p>
            devinlife commetns : 싱글 페이지(포스트)에 제목 밑에 Updated 시간 표기.
            기존에는 read_time 이 표기. read_time -> date 변경
          -->
	<p class="page__date"><i class="fas fa-fw fa-calendar-alt" aria-hidden="true"></i>
		{{ site.data.ui-text[site.locale].date_label | default: "Updated:" }}
		<time datetime="{{ page.date | date_to_xmlschema }}">
			{{ page.date | date: "%B %d, %Y" }}
		</time>
	</p>
	{% endif %}
	{% include page__taxonomy.html %}
</header>{% endraw %}
```

그랬더니 사진과 같이 날짜부분과 굵기가 다르다. 

<2>

굵기와 태그 border 를 조정하기 위해서 어떻게 include 되어있는가를 봤더니, `_layouts/single.html` 에서 `page__taxonomy.html` 를 include하고 있고, 또 `tag-list.html`, `category-list.html` 을 각각 include 하고 있다. 따라서 각 html에 들어가서 `<strong>`을 없애주면 굵기가 원복된다. 

이제 각 border의 크기를 줄여 사이즈에 맞게 바꿔줄 것이다. `_sass/minimal-minstakes/_pages.scss`에 306번째 줄 즈음 

```scss
.page__taxonomy {
  margin-top: $type-size-8;    // 추가
  margin-bottom: $type-size-8; // 추가
  font-size: $type-size-7;
  ...
}

.page__taxonomy-item {
  ...
  padding: 1px 4px;   // 수정
  ...
}
```

이렇게 수정해주고,

```scss
.page__date {
  font-size: $type-size-7;
}
```

자체를 `_pages.scss`  추가해주었다.

<3>

이렇게 나름 정렬을 맞춘 구색을 갖출 수 있다.

> ※ $type-size-7 는 `_variables.scss`에 들어가면 변수로 지정되어 있어서 scss 어디에서든 불러다 쓸 수 있다.
>
> 블로그에 들어가서 크롬 개발자 모드로 일일이 찍어보면서 확인해보면 적용된 태그와 프로퍼티로 scss가 쓰인곳을 찾아서 소스를 수정하는 작업을 거쳤다. 노가다성 작업... 🤮



## 프로필을 포스트 하단으로 옮기기 🏌️‍♀️

왼쪽에 있는 sidebar라고 이름붙여진 프로필을 포스팅 하단으로 내려서 작업하니 아무리해도 붙질 않아서 새로 만들었다.`single.html`에

```html
{% raw %}<footer class="page__meta">
	{% include page__profile.html %}  // 이부분 추가. 새로 만들 것이다.
	{% if site.data.ui-text[site.locale].meta_label %}
		<h4 class="page__meta-title">{{ site.data.ui-text[site.locale].meta_label }}</h4>
	{% endif %}
</footer>{% endraw %}
```

이렇게 include를 이용, 넣어주고 `_includes`폴더에 `page__profile.html` 을 새로 만들어주었다.
