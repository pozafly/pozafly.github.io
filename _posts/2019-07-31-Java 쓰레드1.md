## 쓰레드란?

- 실행되고 있는 프로그램 : 프로세스

- 프로세스에서 처리되는 작업 : 쓰레드

  (즉, 프로그램 안에 프로그램이 프로세스고, 프로세스 안에 프로그램이 쓰레드다.)

 [ex]곰플레이어 : 소리작업, 영상작업 번갈아가면서 실행되고 있다. 근데 주기가 너무 짧기 때문에 동시에 일어나고 있는 것 처럼 느끼는 것. 이 작업 '단위'를 쓰레드라고 함. 프로세스 안에 작업 단위.



### 쓰레드를 만드는 방법

1. Thread 클래스 상속

2. run() 메소드 오버라이딩

3. start() 메소드를 호출(run() 안에 있는 메소드를 실행한다.



[ex1] 단일 쓰레드

```java
class A extends Thread {
    public void run() {
        for (;;) System.out.print("*");
    }
    public static void main(String args[]) {
        A t = new A();
        t.start(); //start시 * 찍었다가 M 찍었다가 순서 없이 찍힘.
        for (;;) System.out.print("M");
    }
}
```

[ex2] 3개의 쓰레드

```java
class A extends Thread {
    String str;
    A(String str) {
        this.str = str;
    }
    public void run() {
        for (;;) System.out.print(str);
    }
}

class B {
    public static void main(String args[]) {
        A t1 = new A("*");
        t1.start();
        A t2 = new A("+");
        t2.start();
        A t3 = new A("#");
        t3.start();
        for (;;) System.out.print("M"); //만약 이게 제일 첫째줄에 있었다면 M만 출력되겠지?
    }
}
```

### 쓰레드 정보 얻기

- void setName(String name) : 쓰레드 이름을 지정하거나 얻을 때 사용. 쓰레드를 생성할 때 이름을 줄 수 있다. 만약 이름을 지정하지 않는다면 새로운 이름이 할당 된다.
- String getName() : 쓰레드 이름 구해와줌.

```java
class A extends Thread {
    String str;
    A(String str) {
        super(str);
    }
    public void run() {
        System.out.println(getName()); //쓰레드1 쓰레드3 쓰레드2 순서 없이 찍힘
    }
}

class B {
    public static void main(String args[]) {
        A t1 = new A("쓰레드1");
        t1.start(); //생성자로 super로 쓰레드 이름 생성
        A t2 = new A("쓰레드2");
        t2.start();
        A t3 = new A("쓰레드3");
        t3.start();
    }
}
```

- activeCurrent() : 현재 쓰레드를 리턴한다.

```java
class A {
    public static void main(String args[]) {
        System.out.println(Thread.currentThread().getName()); //main 만 나옴.
    }
}
```

- static int activeCount() : 실행중인 쓰레드들의 수를 리턴한다.

```java
class A extends Thread {
    public void run() {

    }
}
class B {
    public static void main(String args[]) {
        A t1 = new A();
        t1.start();
        A t2 = new A();
        t2.start();
        A t3 = new A();
        t3.start();
        System.out.println(Thread.activeCount()); //main을 포함해서 4가 찍힌다.

```

- static int enumerate(Thread[] tarray) : 현재 실행중인 쓰레드들의 주소로 배열을 채운다.

```java
class A extends Thread {
    public void run() {}
}
class B {
    public static void main(String args[]) {
        A t1 = new A();
        t1.start();
        A t2 = new A();
        t2.start();
        A t3 = new A();
        t3.start();
        Thread[] array = new Thread[4]; //4개의 빈 쓰레드 배열 만들고
        Thread.enumerate(array); //그 array배열에 각 동작하는 쓰레드의 주소를 넣는다.
        for (Thread t: array) //array별로
            System.out.println(t.getName()); //이름 추출. main, Thread-0, Thread-1, Thread-2 이렇게 나옴.
    }
}
```

- -static Thread currentThread() : 현재 쓰레드들을 리턴한다.

###  쓰레드의 일시정지

Thread클래스의 sleep메서드를 사용하면 쓰레드의 실행을 일시정지 할 수 있다.(밀리세컨드 단위)

```java
class A {
    public static void main(String args[]) throws Exception {
        for (int i = 0; i < 10; i++) {
            System.out.println("Good!");
            Thread.sleep(500); //0.5초 간격으로 10번 Good!을 찍어줌
        }
    }
}
```

- ※현재시간을 long 값으로 구하기.

```java
class A extends Thread {
    public static void main(String args[]) {
        A a = new A();
        long startTime = System.currentTimeMillis(); //System.currentTimeMillis : 현재시간을 밀리세컨드 단위로 구한다.
        try {
            Thread.sleep(5000);
        } catch (InterruptedException e) {}
        long endTime = System.currentTimeMillis();
        System.out.println("쓰레드는 Thread.sleep(5000)동안 쉬게했지만, 사실은 " + (endTime - startTime) + "ms만큼 쉰다."); //쓰레드는 Thread.sleep(5000)동안 쉬게했지만, 사실은 5001ms만큼 쉰다.
    }
}
```

현재 시간을 long 단위로 구할 수 있다. 2019-9-20을 long단위로 구함. 다운로드 파일을 할 때, 파일이름이 없을 경우가 있다. 이럴 때 시간단위로 숫자형을 줘서 이름을 각각 중첩 안되게 할 수 있음.

- yield() : (양보하다)현재 실행중인 Thread객체를 일시적으로 멈춤으로 인해서 다른 쓰레드들을 실행할 수 있도록 해 준다.

```java
class A extends Thread {
    public void run() {
        for (int i = 0; i <= 500; i++) {
            System.out.print("A ");
            //Thread.yield();
        }
    }
}
class B extends Thread {
    public void run() {
        for (int i = 0; i <= 500; i++) {
            System.out.print("B ");
            Thread.yield();
        }
    }
}
class Test {
    public static void main(String args[]) {
        A a = new A();
        a.start();
        B b = new B();
        b.start();
    }
}
```

지금은 B에 yield가 있으므로 A가 초반에 많이 찍힌다. 영상같은 쓰레드 작업에서는 yield가 있을 수 없다. 영상이 소리에 밀릴 수 없으니까. `가비지처리` 하는 것 같은데 사용됨.





#### 특성

- 쓰레드는 영상을 예로 들어서 생각해보면, 영상 나오고 소리 나오고 이런 것이 반복적으로 서로 번갈아가면서 계속 일어나는 것. 작업 하나 하나를 쓰레드라고 한다.

- 프로세스 : 실행되고 있는 프로그램

- Context Switching(컨텍스트 스위칭) : 프로그램이 여러개가 돌아가고 있는데 메모리에 전부다 안올라간다. 메모장을 사용하고 있다면 메모장에 관련된 것만 메모리에 올라간다. 나머지는 메모리에 없음. 바꾸는 순간 또 다른 프로세스가 메모리에 올라감. 이때 자원을 바꾸고 싶을 때는 'interrupt'가 발생한다. 
- 쓰레드가 한개가 고장나면 프로그램이 동작하지 않을 수 있다.

### join

: join의 역할은, `쓰레드가 끝날 때 까지 기다린다`는 의미. 왜냐하면 중간에 System.out.println("끝"); 이 실행되어서는 안되니까. 하지만 만약 join문장이 없었다면 시작과 동시에 System.out.println("끝"); 이 실행될 가능성이 있다. 무조건 제일 마지막에 찍히지는 않음.

```java
class A extends Thread {
    String str;
    public A(String str) {
        this.str = str;
    }
    public void run() {
        for (int i = 0; i < 5; i++) {
            System.out.println(str); //생성자 이름 불러옴.
            try {
                sleep((int)(Math.random() * 1000)); //랜덤 시간만큼 기다림
            } catch (Exception e) {}
        }
    }
}
class JoinTest {
    public static void main(String args[]) {
        A a1 = new A("A쓰레드");
        A a2 = new A("B쓰레드");
        a1.start();
        a2.start();
        try {
            a1.join(); //a1 쓰레드가 끝날때 까지 기다림
            a2.join(); //a2 쓰레드가 끝날때 까지 기다림
        } catch (Exception e) {
            e.printStackTrace();
        }
        System.out.println("끝"); //마지막에 끝이 찍힌다.
    }
}
```

[ex2] 1-50, 51-100 까지 합을 구하는 쓰레드를 각각 만들고 합계를 찍어보자.

```java
class A extends Thread {
    int n1, n2;
    int sum = 0;
    A(int n1, int n2) {
        this.n1 = n1;
        this.n2 = n2;
    }
    public void run() {
        for (int i = n1; i <= n2; i++) {
            sum += i;
        }
    }
}
class B {
    public static void main(String args[]) throws Exception {
        A a1 = new A(1, 50);
        A a2 = new A(51, 100);
        a1.start();
        a2.start();
        a1.join();
        a2.join(); //join이 없다면 합 : 0 이 나온다. 이미 실행되기 때문에.
        System.out.println("합 : " + (a1.sum + a2.sum)); //5050
    }
}
```
