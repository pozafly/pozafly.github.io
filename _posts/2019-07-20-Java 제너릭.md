## Generic이란?

* `포괄적인` 이라는 의미. 
* 클래스 옆, 이름 옆, <>를 선언 후, 안에 아무 문자나 넣는다. 이때 보통 T(Type)를 사용하게 됨. 자료형.

```java
[ex]
class A<T>{  //제너릭 클래스. T는 타입이라는 의미로 사용한다.
	T t;
	void setValue(T t){ this.t = t; }
	T getValue(){ return t; }
}
class B{
	public static void main(String args[]){
		A<String> o1 = new A<String>(); //T 자리에 String을 사용했기 때문에 제너릭 클래스 타입은 String.
		o1.setValue("aaa");                  //String 타입.
		A<Integer> o2 = new A<Integer>();
		o2.setValue(100);
	}
}
```

※ `Wrapper클래스` : 자바 기본형에 대응되는 클래스(포장지 클래스)

- int   --> java.lang.Integer

- char   --> java.lang.Character

- double  --> java.lang.Double

- boolean --> java.lang.Boolean

### 제너릭 메서드

: 메서드에 제너릭 형식을 붙이는 것. 제너릭 클래스로 선언하지 않았지만, 제너릭 메서드를 만들고 싶을 때, 메서드 앞에 제너릭 타입을 붙이면 된다.

-메서드 : 

```java
[ex]
import java.util.*;
class Box {}
class A {
    <T> T getValue(ArrayList<T> list, int index) {  //제너릭 메서드
        return list.get(index);
    }
}
class B {
    public static void main(String args[]) {
        ArrayList < String > arraylist1 = new ArrayList < String > ();
        ArrayList < Box > arraylist2 = new ArrayList < Box > ();
        arraylist1.add("aa");
        arraylist1.add("bb");
        arraylist2.add(new Box());
        arraylist2.add(new Box());
        A obj = new A();       //제너릭 클래스가 아님
        String v1 = obj.getValue(arraylist1, 1);
        Box v2 = obj.getValue(arraylist2, 0);
        System.out.println(v1); // bb
        System.out.println(v2); // Box@주소
    }
}
```

### 제너릭 배열 선언

```java
[sy]
T[] array = (T[])(new Object[n]);   //여기서 n은 몇개 방을 만들건지.

[ex]
class Course<T>{
	String name;
	T[] students;
	Course(String name, int students){
		this.name = name;
		//this.students = new T[] students();   //제너릭 타입에서 이렇게 배열 만드는 것은 불가.
		this.students = (T[]) (new Object[students]); //이게 제너릭 배열 선언임.
	} 								//int로 들어오는 students 갯수만큼 배열선언함.
}
```

### 제한된 타입 파라미터 \<T extends 최상위 타입>

- 타입 파라미터에 지정되는 구체적인 타입을 제한할 필요가 종종 있음.

  [ex] public static \<T extends Number> int compare(T t1, T t2)

  라는 메서드는 Number 타입의 하위 클래스만 들어올 수 있음. 그냥 T만 했다면 String 이건 뭐건 다 들어왔겠지.

```java
[ex]
	static <T extends Animal> T getAnimal (ArrayList<T> list, int index){
		return list.get(index);
	}
```

->원래는 매개변수 ArrayList\<T> 부분에 \<T extends Animal>이 들어와야한다고 생각하지만, 아니다. 메서드 앞에 명시해주고 매개변수쪽 제너릭에는 앞에 선언된 타입을 따른다.

<br/>

### 제너릭 와일드카드 타입(<?>, <? extends ...>, <? super...>)

\<T> 대신 <?>를 사용한다. T라면 메소드가 시작되면 타입이 하나로 정해져 있고, ? 표는 어떤 형식이 와도 상관없는 것. 

<?> 와일드카드 타입은 제네릭 타입을 `매개값`이나 `리턴타입`으로만 사용한다.

- \<T>와 \<?>의 차이점 :

\<T>는 상속구조일 때, 형변환 불가능. \<?> 는 상속구조일 때, 형 변환처럼 담기 가능. 즉, \<?>는 \<Object> 다.

 <br/>

1. \<?> : 제한없음

2. \<? extends 상위타입> : 상위 클래스 제한

3. \<? super 하위타입> : 하위 클래스 제한

|        | Person |             |
| ------ | ------ | :---------: |
| Worker |        |   Student   |
|        |        | HighStudent |

이런 상속 관계라고 했을 때,

- void registerCourse(Course<?> course) -> 매개변수로 Person, Worker, Student, HighStudent모두 들어올 수 있음.

- void registerCourse(Course<? extends Student> course) -> Student, HighStudent 만 가능.

- void registerCourse(Course<? super Woker> course) -> Worker, Person 만 가능.

```java
[ex]
import java.util.*;
class Animal {}
class Dog extends Animal {}
class Cat extends Animal {}
class A {
	static void test(ArrayList<? extends Animal> list){
	}
    public static void main(String args[]) {
        ArrayList<Dog> list1 = new ArrayList<Dog> ();
        ArrayList<Cat> list2 = new ArrayList<Cat> ();
        test(list1);
        test(list2);
    }
}
```
