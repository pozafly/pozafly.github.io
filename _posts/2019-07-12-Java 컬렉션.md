## Collection이란?

*  배열의 단점을 보완한 것(순서대로, 중복가능, 크기가 졍해져 있는 것). 자료를 저장하는 방식에 따라 클래스로 만듦. 이 클래스의 묶음을 컬렉션이라고 한다.
* 자료구조를 객체모델링한 것.(Car클래스 --> 자동차를 객체모델링한 것.) 자료구조란? 자료를 저장하는 방식.

### 종류

1. 주머니 : Set(HashSet) -     순서없음, 중복없음
2. 배열 : List(ArrayList) - 순서있음, 중복가능,     사용빈도 상.
3. 사전 : Map(HashMap) - 순서없음, 키 중복불가능. 키와 값을 한쌍의 요소로 가지는 컬렉션.

## HashSet

-메서드 : 

- ★add() : 값을 넣는다.

- ★size() : HashSet 길이 반환.

-주머니를 생각하면 된다. 순서가 없음. 그리고 중복이 불가.

-확장for문으로 출력. 

```java
[ex]
import java.util.*;
class A {
    public static void main(String args[]) {
        HashSet < String > set = new HashSet < String > ();
        set.add("aa");
        set.add("bb");
        set.add("cc");
        set.add("aa");
        System.out.println(set); //aa,bb,cc 중복 불가능하기 때문에 3개만 나옴.
    }
}
```

- ★iterator() : 반복자

시그너처 : public Iterator\<E\> iterator()

```java
[ex]
import java.util.*;
class A{
	public static void main(String args[]){
		HashSet<Integer> set = new HashSet<Integer>();
		set.add(10);
		set.add(20);
		set.add(30);
		Iterator<Integer> iter = set.iterator();
		while(iter.hasNext()){
			int n = iter.next();
			System.out.println(n);
		}
	}
}
```

### ※ 이터레이터에 대해서

- 이터레이터는 '반복'이라는 뜻을 가지고 있다.(HsahSet에서는 iterator()를 사용해, 배열의 확장 for문 출력하는 것과 같이 할 수 있음.)

- `hasNext()` 메서드는 다음 자료가 있나 검사하는 것. boolean형.

- `Next()` 메서드는 다음 자료를 불러와 변수에 집어 넣음.

근데, iterator안쓰고, 확장for문 써도됨.

- Iterator는 확장 for문과 같다. but, 확장for문은 지울 수 있는게 없다. for문에서 `remove()` 사용하려면, Iterator 추천.

## TreeSet

HashSet과 기능이 같지만, 자동 오름차순 해주는 class. TreeSet은 자동 오름차순 정렬이다. 내림 차순으로 하고 싶을 때는 어떻게 해야하는가?

다른 생성자 : TreeSet(Comparator<? super E> comparator)

```java
[ex]
import java.util.*;
class A{
	int a;
	A(int n){a = n;}
}
class B{
	public static void main(String args[]){
		TreeSet<A> set = new TreeSet<>();
		set.add(new A(100));
	}
}
```

이라고 했을 때, ClassCastException 이 뜬다. TreeSet은 정렬 기능이 포함된 Set인데 class A는 정렬할 수 없기 때문. 이때, Interface `Comparable<T>` 을 사용함. 메소드는 한개다. public int compareTo(T o) 이 메소드를 오버라이딩 해서 비교할 것을 정해주어야 함.

```java
public int compareTo(T o){
	현재 객체가 크면 양수값 리턴
	현재 객체가 작으면 음수값 리턴
	비교했더니 같으면 0
}
```

Integer 클래스를 열었더니

implements Comparable<Integer> 라고 적혀있다. 비교할 수 있는 객체다.

근데 우리가 만든 Person객체는 비교할 수 있는 객체가 아님. 근데 여기에 Comparable을 구현해놓으면 된다.

```java
[ex]
import java.util.*;
class Person implements Comparable<Person>{  //구현클래스
    int age;
    Person(int a) {age = a;}
    public String toString() {
        return "나이:" + age;
    }
	public int compareTo(Person anotherPerson){
		return age-anotherPerson;  //오름차순
		//return anotherPerson-age;  이건 내림차순
	}
}
class A {
    public static void main(String args[]) {
        TreeSet < Person > set = new TreeSet < > ();
        set.add(new Person(10));
        set.add(new Person(5));
        set.add(new Person(100));
        System.out.println(set); // 5 10 100
    }
}
```

이렇게 하면된다.

※ Comparable과 Comparator의 차이점 : Comparable 은 우리가 만든 클래스를 재정의 하는 용도로 사용되고, Comparator 는 Integer클래스와 같이 이미 정의된 클래스를 더 쉽게 사용하는 것임.

근데 더 쉽게 하고 싶다. Comparator사용.

```java
import java.util.*;
class DescComparator implements Comparator<Integer>{ //인터페이스 중 default, static 메서드들 무시 가능.
	public int compare(Integer o1, Integer o2){
		return o2-o1;
	}
}
class A {
    public static void main(String args[]) {
        TreeSet < Integer > set = new TreeSet < > (new DescComparator());
        set.add(10);
        set.add(-1);
        set.add(5);
        System.out.println(set); // 10 5 -1
    }
}
```

## ArraryList

HashSet과 메서드가 같다. ★`add()`,` size()`. 그리고, ★`get(int index)`라는 메서드가 있음. ArrayList는 순서가 있으므로 index자리에 방 번호를 입력한다. 반환형은 제너릭. `remove(int index)` : 방번호 값이 지워짐.

```java
[ex]
import java.util.*;
class Person{
	String name;
	Person(String n){name = n;}
}
class A{
	public static void main(String args[]){
		Person p1 = new Person("홍길동");
		Person p2 = new Person("김길동");
		Person p3 = new Person("박길동");
		
		ArrayList<Person> list = new ArrayList<>();
		list.add(p1);list.add(p2);list.add(p3);
		for(Person p : list)
			System.out.println(p.name);
	}
}
```

※ 팁

1. ArrayList() : ArrayList는 기본적으로 10개 방을 만든다. 11개 째에 10개가 더 추가됨.(Constructs an empty list with an initial capacity of ten.)

2. ArrayList(int initialCapacity): int initialCapacity에 용량을 지정해줄 수 있다. 그러면 방을 다시 만드는 작업을 안해도 되니 메모리 아낄 수 있음.

- ArrayList에서는 중간에 2번방 값이 없어졌다라고 가정하면 앞 3번방 값이 2번방으로 끌려온다.

- Vector와 ArrayList 차이 거의 없다. 하지만, 동기화와 관련있음.

- ArrayList나 LinkedList는 사용법은 같다. 하지만 메소드별 속도차이가 있음. 하지만 DB단에서 속도를 결정하기 때문에 어떤 것을 사용해야하는지 고민할 필요 없음. 거의 차이 없다.

## HashMap

HashMap<K,V> : k는 key를 말함. v는 value를 말함.

### 메서드

- put(K,V) : 집어넣는다 key와 value를.

- public Set<K> set keySet() : key값을 set이라는 변수에다 담는다.

- public V get(Object key) : key값을 넣으면, value값 형 리턴받는다.

- public boolean containsKey(Object key) : key가 있으면 true, 틀리면 false를 반환.

```java
[ex]
import java.util.*;
class Person{
	String name;
	Person(String n){name = n;}
}
class A{
	public static void main(String args[]){
		Person p1 = new Person("홍길동");
		Person p2 = new Person("김길동");
		Person p3 = new Person("박길동");
		Person p4 = new Person("최길동");
		Person p5 = new Person("이길동");
		Person p6 = new Person("강길동");
		
		ArrayList<Person> list1 = new ArrayList<>();
		ArrayList<Person> list2 = new ArrayList<>();
		ArrayList<Person> list3 = new ArrayList<>();
		list1.add(p1);list1.add(p2);list1.add(p3);
		list2.add(p4);list2.add(p5);
		list3.add(p6);
		
		HashMap<String, ArrayList<Person>> map = new HashMap<>(); //value값으로 ArrayList<Person>형이 사용됨.
		map.put("인사팀", list1);
		map.put("회계팀", list2);
		map.put("물류팀", list3);
		
		Set<String> key = map.keySet(); //key는 String형 이므로 set다음에 String이 사용됨.
		for(String str : key){ // key크기 만큼 돈다
			ArrayList<Person> list = map.get(str); //돌 때마다 list를 만들어서 매번 str에 찍히는 key값을 보고 list주소 값을 넣고 
			System.out.print(str+" : ");
			System.out.println(list.size()); //사이즈를 구한다.
		}
	}
}
```
