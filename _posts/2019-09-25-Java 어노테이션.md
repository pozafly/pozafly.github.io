## 어노테이션이란?

- 어노테이션(Annotation)은 메타데이터(metadata)라고 볼 수 있다. 메타데이터란 어플리케이션이 처리해야 할 데이터가 아니라, 컴파일 과정과 실행 과정에서 코드를 어떻게 컴파일하고 처리할 것인지를 알려주는 정보.

- 용도

  1. 컴파일러에게 코드 문법 에러 체크하도록 정보제공. (ex. @Override : 오버라이딩 제대로 안되면 컴파일 에러.)

  2. SW개발 툴이 빌드나 배치 시 코드를 자동으로 생성할 수 있도록 정보제공.

  3. 실행 시(런타임 시) 특정 기능을 실행하도록 정보제공.

<br/>

<br/>

### 선언방법

: public @interface xxx 이렇게 선언한다. 그냥 인터페이스인데, @를 앞에 붙임.

```java
[sy]
@Target(value=[ElementType])  //어느것 위에 쓸 것인지?
@Retention(value=[RetentionPolicy])  //범위는 어디까지 건들일 것인지?
public @interface ABC{  //어노테이션 이름과 실행
	...
}
```

<br/>

### 적용대상(@Target)

 : 어노테이션을 어디 위에다 붙일 것인지?  java.lang.annotation.ElementType 열거 상수로 정의되어 있음.

- `TYPE` : 클래스, 인터페이스, 열거타입
- `ANNOTATION_TYPE` : 어노테이션
- `FIELD` : 필드(멤버변수)
- `CONSTRUCTOR` : 생성자
- `METHOD` : 메소드
- `LOCAL_VARIABLE` : 로컬변수
- `PACKAGE` : 패키지

<br/>

### 유지정책(@Retention) 

 : 사용 용도에 따라 @AnnotationName을 어느 범위까지 유지할 것인지 지정해야 한다. 쉽게 설명하면 소스상에서만 유지할 건지, 컴파일된 클래스까지 유지할 건지, 런타임 시에도 유지할 건지를 지정해야 한다. 어노테이션 유지 정책은 java.lang.annotation.RetentionPolicy 열거 상수로 정의되어 있음.

 

- `SOURCE` : 소스 상에서만 어노테이션 정보 유지. 소스 코드 분석할 때만 의미. 바이트 코드 파일에는 정보가 남지 않음(즉, class파일까지 안간다).

- `CLASS` : 바이트 코드 파일까지 어노테이션 정보 유지. 하지만 리플렉션 이용해서 어노테이션 정보를 얻을 수 없음(잘 이용 안함).

- `RUNTIME` : 바이트 코드 파일까지 어노테이션 정보를 유지, 리플렉션을 이용, 런타임 시 어노테이션 정보를 얻을 수 있음.

 

[ex]

JavaStandard버전에서는 Annotation Type Override를 보면 @Retention(value=SOURCE) //소스분석만 해주겠다는 뜻. 컴파일 할 때만 보겠다.

JavaEE web버전에서 Annotation Type WebServlet어노테이션을 보면 이렇게 되어있다. @Retention(value=RUNTIME) //즉, runtime시 사용하겠다는 뜻.

## 한번 만들어보자.

※public \<A extends Annotation> A getAnnotation(Class\<A> annotationClass)

: 어노테이션을 얻어와라. A(제너릭) 형태로. 하지만, \<A extends Annotation> A는 어노테이션이 아니면 안된다.

```java
import java.lang.annotation.*;
@Target(value=ElementType.TYPE)  //class에 적용가능. 근데 여기서 상수이기 때문에, ElementType.이 필요함.
@Retention(value=RetentionPolicy.RUNTIME)  //RetentionPolicy.도 마찬가지.
@interface ABC{
	String a();  //반환형이 String인 a 추상메서드
	int b();  //반환형이 int인 b 추상메서드
	int[] c();
}
@ABC(a="abc", b=100, c={200, 300})  //@Override는 추상메서드가 없었어서 그냥 적어줬지만, 우리가 만든 @ABC는 추상메서드가 있으므로 괄호안에 리턴값을 적어준다. 이 괄호 안에 값이 위의 추상메서드를 오버라이딩 해놓은 것이다.
class Main{
	public static void main(String args[]){
		Class<Main> classObj = Main.class;   //Main클래스를 classObj라는 변수에 집어넣었다.
		ABC obj = classObj.getAnnotation(ABC.class);   //ABC어노테이션을 객체생성해서 반환
		System.out.println(obj.a());  //추상메서드를 부른다. 오버라이딩 된 것을.
		System.out.println(obj.b());  //100 출력.
		System.out.println(obj.c()[0]);  //0번 방에 있는 200 출력.
	}
}

//단 @ABC()안에 배열을 적어줄 때 값이 1개면 {} 중괄호 생략가능하다.
```

#### enum 활용

```java
import java.lang.annotation.*;
@Target(value=ElementType.TYPE)
@Retention(value=RetentionPolicy.RUNTIME)
@interface ABC{
	DEF[] d();  //enum의 배열 추상메서드
}
enum DEF{AA, BB}
@ABC(d={DEF.AA, DEF.BB})  //d()오버라이딩
class Main{
	public static void main(String args[]){
		Class<Main> classObj = Main.class;
		ABC obj = classObj.getAnnotation(ABC.class);
		DEF[] array = obj.d();   //array에 넣고
		System.out.println(array[0]);  //출력
		System.out.println(array[1]);
	}
}
```

#### 메서드에 붙여보자

```java
import java.lang.annotation.*;
import java.lang.reflect.*;    //Method를 부르기 위해 리플렉트 패키지 필요
@Target(value=ElementType.METHOD)  //메서드 상수를 사용.
@Retention(value=RetentionPolicy.RUNTIME)
@interface ABC{
	String a();
}
class DEF{
	@ABC(a="aaa")   //메서드 위에 붙였음. 추상메서드 a() 오버라이딩 함.
	void abc(){
		System.out.println(1);
	}
}
class Main{
	public static void main(String args[]) throws Exception{
		Class<DEF> classObj = DEF.class;   //DEF클래스를 받아옴.
		Method m = classObj.getDeclaredMethod("abc");  //DEF클래스의 abc()메서드를 따와서 m에 저장.
		ABC obj = m.getAnnotation(ABC.class);  //abc()메서드의 어노테이션을 얻어라. ABC.class어노테이션을.
		System.out.println(obj.a());  //오버라이딩 된 어노테이션ABC의 메서드 a()를 불러라.
		m.invoke(new DEF());  //DEF에 있는 가져온 abc메서드를 바로 호출함. invoke는 메서드 호출하는 메서드다. 즉, 1이 찍힘.
	}
}
```

#### Default - 추상메서드 옆에 default 값을 줄 수 있다.

```java
import java.lang.annotation.*;
import java.lang.reflect.*;
@Target(value=ElementType.METHOD)
@Retention(value=RetentionPolicy.RUNTIME)
@interface ABC{
	String a() default "bbb";  //디폴트 값을 줬다.
}
class DEF{
	@ABC()  //default가 있으므로 괄호 안에 아무것도 주지 않으면 디폴트 값이 나온다.
	void abc(){
		System.out.println(1);
	}
}
class Main{
	public static void main(String args[]) throws Exception{
		Class<DEF> classObj = DEF.class;
		Method m = classObj.getDeclaredMethod("abc");
		ABC obj = m.getAnnotation(ABC.class);
		System.out.println(obj.a());   //bbb 출력됨.
	}
}
```

### @SuppressWarnings

: 경고가 뜨는데 경고가 거슬릴 때 사용하는 어노테이션.

```java
import java.util.*;
class A{
	@SuppressWarnings(value={"rawtypes", "unchecked"})
	//@SuppressWarnings({"rawtypes", "unchecked"})이렇게도 가능. value가 기본 추상메서드라 생략해줬다.
	public static void main(String args[]){
		ArrayList list = new ArrayList();  //ArrayList는 제너릭이지만 제너릭을 써주지 않아서 경고가 뜰 것이다.
		list.add(100);
	}
}
```

@SuppressWarnings 어노테이션에 들어가봤더니 추상메서드로 value가 있었고 String[] 형태였다. 따라서 그곳에는 경고 종류가 배열형식으로 들어감. 따라서 오버라이딩 해줘야 했기 때문에 rawtypes과,  unchecked를 써놓은 것임.
