## 람다식이란?

- 내부 무명 클래스를 최대한 축약해서 표현하는 방법. 함수지향 언어.

   코드가 매우 간결해지고, 컬렉션의 요소를 필터링 하거나 매핑해서 원하는 결과를 쉽게 집계할 수 있다.

```java
[sy]
= () -> {}
```

#### 람다식에서의 생략 

1. {}중괄호 안에 들어가는 코드는 한줄 일 때 생략가능하다.

2. 매개변수 한개라면 ()도 생략가능. but 매개변수가 없거나 두개 이상일 경우 생략 불가.

3. return문이 있다면 return 단어도 생략 가능함.

4. 뻔해서 자료형도 필요 없다.

### 내부무명클래스 -> 람다식으로 변형

- 내부무명클래스

```java
interface A {
    void abc();
}
class B {
    public static void main(String args[]) {
        A o = new A(){
			public void abc(){
				System.out.println(100);
			}
		};
        o.abc(); // 100
    }
}
```

이걸 람다식으로 바꿔보면?

- 람다식

```java
A o = new A(){
	public void abc(){         //이부분을 지워버린다.
		System.out.println(100);
	}
};
```

그러면
A o = (){ System.out.println(100); };  여기 사이에 '->' 를 넣어줌
A o = () -> { System.out.println(100); };   근데 한줄이니까 중괄호도 필요 없음.
A o = () ->  System.out.println(100);



- 람다식은 인터페이스에도 적용된다. 그리고 메서드는 딱 한개만 가능하다. 전문용어로 펑셔널 인터페이스(functional interface).

  펑셔널 인터페이스는 인터페이스에 추상메서드가 딱 한 개 있다는 뜻이다.

```java
@FunctionalInterface
interface A {
    void abc();     //메서드 딱한개. 
}
class B {
    public static void main(String args[]) {
        A o=()-> System.out.println(100); //오버라이딩
        o.abc(); // 100
    }
}
```

ActionListener o = e -> System.out.println(e.getSource());

이렇게.



※  단, interface이지만 메서드가 한개인 functional interface가 아닌경우 API를 보면 구현인터페이스가 따로 있다. 람다식을 사용하고 싶을 때는 구현객체를 사용하면 된다.

즉, interface를 implements받는 메서드가 한개인 interface를 직접 선언해서 사용. Thread 인터페이스의 경우 Runnable 인터페이스가 있다.

Runnable인터페이스에는 run() 메서드가 단 한개뿐이다.
