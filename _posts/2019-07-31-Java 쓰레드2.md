### Interrupt

- 예외(InterruptedException) 이용 : 멀티쓰레드를 사용한 자바 프로그램에 익숙해지면 메서드에 throws InterruptedException이 있을까 없을까를 의식하게 된다. 즉, 예외가 있는 메서드는 시간이 걸릴지 몰라도 취소 할 수 있는 메서드이다. 자바의 표준 클래스 라이브러리에서 throws InterruptedException이 있는 대표적인 메서드는 3가지가 있다. 

java.lang.Object 클래스의 wait()

java.lang.Thread 클래스의 sleep() --> 정해진 시간만큼 정지

java.lang.Thread 클래스의 join() --> 해당 쓰레드가 종료될 때까지 다른 쓰레드 정지

이처럼 이 세가지 메서드는 \[interrupt/notify/notifyAll](인터럽트 메서드)되기까지, 지정한 시간만큼, 지정한 쓰레드가 종료될 때까지 확실히 시간이 걸리는 메서드가 되어 있다. 이렇게 시간이 걸리는 것을 interrupt메서드를 이용해 취소할 수 있다.

<br/>

- interrupt() : 쓰레드가 멈춰있을 때 깨운다. 다시 동작하도록 하는 신호를 보내줌. 

단, interrupt가 발생하면 InterruptedException이 발생하게 되어있다.

```java
class Echo extends Thread {   //Thread를 상속받았다.
	private int waitTime;
	public Echo(String word, int waitTime) {
			super(word);   //쓰레드 이름 설정
			this.waitTime = waitTime;
	}
	public void run() {
		for(int i=0; i<10; i++) {
			System.out.print(getName()+">"+i+"\t");
			try {
				sleep(waitTime);
			}catch (InterruptedException e){
				System.out.println(getName()+":인터럽트발생\t");
			}
		}
	}
	public static void main(String[] args) {
		Echo e1 = new Echo("Foo",  1000);   //쓰레드 이름, 몇초간 sleep할껀지.
		Echo e2 = new Echo("Bar",  1000);
		e1.start(); e1.interrupt(); e2.start(); e2.interrupt();
		//e1 쓰레드가 동작, e1에 인터럽트 발생시킴, e2 동작, e2 인터럽트. 즉, 한번 깨우고 다음에는 그냥 내부 쓰레드가 동작한다.
	}
}


[결과]
Foo>0  Foo:인터럽트발생  Foo>1   
Bar>0  Bar:인터럽트발생  Bar>1   
Foo>2   Bar>2   Foo>3   Bar>3
Foo>4   Bar>4   Foo>5   Bar>5   Foo>6   Bar>6
Foo>7   Bar>7   Foo>8   Bar>8   Foo>9   Bar>9
```

이런식으로, 인터럽트는 1번만 발생하는데, 실행되자마자 5초를 재우는데 interrupt()메서드로 바로 깨움. 그다음엔 5초 간격으로 실행된다.

### Producer/Consumer패턴(생산자 소비자 패턴)

- 생산자 소비자 패턴은 Producer/Consumer모델의 경우로 특정한 자원을 생성하는 Producer가 자원을 만들어 낼때까지 사용자인 Consumer는 '대기상태'에 있게 되므로 적은 연산력을 소모하게 된다는 장점을 가지고 있다. 이런 모델에 적합한 패턴이 sleep/interrupt , wait/notify가 있다. sleep/interrupt 패턴이 깨워야 하는 다음 쓰레드를 기억해야 하는 번거로움이 있으므로 wait/notify가 더 자주 사용된다.

 

*아래 예제는 3개의 쓰레드가 서로 번갈아서 동작할 수 있도록 만들었다.

세개의 쓰레드를 sleep상태로 전환한 후 첫번째 스레드에 대해 interrupt메서드를 호출해주면 첫번재 스레드가 깨어나서 동작하게 된다. 첫번째 쓰레드는 필요한 작업을 한 다음에 두번째 스레드를 깨우게 되고 세개의 스레드가 서로 번갈아가며 작업을 하게 되는 구조이다.

[ex1]

```java
class Echo extends Thread {
    Thread t;
    Echo(String word) { super(word); }  //쓰레드 이름 설정
    public void setNextThread(Thread t) { this.t = t; }  //다음 쓰레드의 주소를 받아놓는다.
    public void run() {
        for(int i = 0; i < 10; i++) {
            try {
                sleep(3000);
            } catch (InterruptedException e) {}
            System.out.print("Thread" + getName() + ">" + i + "\t");
            t.interrupt();   //다음 실행하는 쓰레드에게 인터럽트를 걺. 즉, sleep(3000); 이 무시되고 바로 실행됨.
        }
    }
    public static void main(String[] args) {
        Echo e1 = new Echo("일");
        Echo e2 = new Echo("이");
        Echo e3 = new Echo("삼");
        e1.setNextThread(e2);  //e1의 다음쓰레드는 e2
        e2.setNextThread(e3);  //e2의 다음쓰레드는 e3
        e3.setNextThread(e1);  //e3의 다음쓰레드는 e1
        e1.start(); e2.start(); e3.start();
        e1.interrupt();  //첫 쓰레드를 깨운다.
    }
}
```

[ex2] 생산자는 3개를 생성하고, 소비자는 물건을 소모하면서 서로 자원을 사용하는데, 생산자가 3개를 생성했다면 잠을 자러가게 되고 소비자가 물건이 0이라면 자러간다. 자러가는 동시에 상대방을 깨우는 interrupt메서드를 호출하게 된다.

```java
import java.util.*;
class Producer extends Thread {  //생산자. 생산자가 물건을 생산해야 소비자가 소비할 수 있다.
    Resource source;  //창고 주소
    public Producer(String n, Resource s) {  //창고 주소를 받아왔음
        super(n);
        source = s;
    }
    public void run() {
        for(char ch = 'A'; ch < 'J'; ch++) {  //A B C D E F G H I 순
            System.out.println(getName() + "가 제품[" + ch + "]를 생산했다.");
            source.put(ch);  //물건(ch)을 창고에 집어넣는다.
        }
    }
}
class Consumer extends Thread {
    Resource source;
    public Consumer(String n, Resource s) {
        super(n);
        source = s;
    }
    public void run() {
        char getch = ' ';  //제품명
        for(char ch = 'A'; ch < 'J'; ch++) {
            getch = source.get();
            System.out.println(getName() + "가 제품[" + getch + "]를 소비했다.");
        }
    }
}
class Resource {    //창고
    Vector<Character> vec;   //백터(ArrayList랑 비슷)에 Character(char : 문자1개)형태로 저장
    static Producer p;
    static Consumer c;
    public Resource() {
        vec = new Vector<Character>();   //백터 하나 생성
    }
    public void put(char ch) {  //put메서드(Producer가 사용한다)
        while(vec.size() == 3) {   //백터창고 사이즈가 즉, 물건이 3개라면(창고가 다 찼다면)
            try {
                Thread.sleep(5000);  //5초간 쉰다
            } catch (InterruptedException e) {}
        }
        vec.add(ch);   //창고에 넣는다.
        System.out.println("창고에 제품[" + ch + "]가 들어왔다. 재고수량:" + vec.size());
        c.interrupt();   //자고있는 소비자를 깨운다.
    }
    public char get() {  //get메서드(Consumer가 사용한다)
        while(vec.size() == 0) {   //창고에 물건이 없다면
            try {
                Thread.sleep(5000);   //5초간 쉰다
            } catch (InterruptedException e) {}
        }
        char ch = vec.remove(0);   //창고에 물건이 있다면 소비자가 소비한다. 물건을 없앤다.
        System.out.println("창고에 제품[" + ch + "]가 나갔다. 재고수량:" + vec.size());
        p.interrupt();   //자고 있는 생산자를 깨운다.
        return ch;   //물건을 리턴함.
    }
    public static void main(String args[]) {
        Resource r = new Resource();   //창고하나 건설
        p = new Producer("생산자", r);   //생산자,  창고의 주소를 넘겨줌
        c = new Consumer("소비자", r);   //소비자,  창고의 주소를 넘겨줌
        p.start();  //가동
        c.start();  //가동
    }
}
```

### 자원 공유와 동기화 메서드

- 모니터 : 어떤 쓰레드가 락이 걸린(synchronized 코드를 포함하는) 개체의 synchronized 코드 부분을 실행하기 위해서는 거기에 접근할 수 있는 권한이 필요한데, 이러한 용도로 사용되는 '열쇠'를 모니터(monitor)라 한다.

![image](https://user-images.githubusercontent.com/59427983/106706370-ef0df780-6632-11eb-96fa-96e889331a3a.png)

예를들어 위 사진과 같이 은행계좌에서 아래와 같은 일이 발생할 수 있다. 도중에 작업이 끝나지 않았는데 인출을 하게 되는 경우. 즉 오라클에서 트랜잭션(락) 기능이 구현 되지 않은 경우 이런 경우가 발생할 수 있다. 따라서 synchronized 가 필요.

- synchronized

```java
class Myclass{
	synchronized public void method1(){
		// 메서드의 코드..
	}
	synchronized public void method2(){
		// 메서드의 코드..
	}
	public void method3(){
		// 메서드의 코드..
	}
}
```

Myclass의 세 메서드 중 두개는 synchronized로 선언되어 있으므로 이 클래스의 모든 객체에서 이들 메서드는 동시에 하나만 실행될 수 있다.

![image](https://user-images.githubusercontent.com/59427983/106706488-21b7f000-6633-11eb-851a-15f69ebdb06e.png)

즉, synchronized 가 선언된 메서드에는 그 작업이 다 끝나기 전까지 다른 쓰레드가 접근할 수 없다는 뜻임. 자바버추얼머신이 스케줄링을 하는데 그 쓰레드 끝나기 전까지 대기목록에 넣어놓음.

```java
class ManyClient {
    StringBuffer str = new StringBuffer("ABCDE");
    public static void main(String args[]) throws InterruptedException {
        ManyClient mc = new ManyClient();  //자신의 객체 생성
        mc.makeThread("가나다라마");
        mc.makeThread("abcde");
        Thread.sleep(3000);
    }
    public void makeThread(String s) {  //쓰레드를 만드는 메서드
        Thread t = new Thread() {  //새 쓰레드를 만드는데 쓰레드는 interface임. 내부무명클래스 사용.
            public void run() {  //오버라이딩 하는 메서드는 run밖에 없다. 따라서 람다 가능해짐.
                addStr(s);   //어쨌든 이 쓰레드는 위에 메인에서 만들어진 쓰레드에 매개변수로 s를 저장한다.
            }
        };
        t.start();
    }
    public void addStr(String other) {  //s를 여기서 받는다.
        for(int i = 0; i < other.length(); i++) {  //s의 길이만큼 반복
            try {
                str.append(other.charAt(i));   //첫 StringBuffer("ABCDE")에 붙일 것이다. i번째 있는 문자열을.
                System.out.println("Str : " + str);
                Thread.sleep(100);  //0.1초 쉰다
            } catch (Exception e) {}
        }
    }
}

Str : ABCDE가a나b다c라d
Str : ABCDE가a나b다c라d마
Str : ABCDE가a나b다c라d마e
```

### Runnable 인터페이스

- Runnable 인터페이스는 Thread 등의 인터페이스의 구현객체가 되기위한 인터페이스이다. 즉, Thread 에서 람다식을 쓰고 싶은데 Thread는 펑셔널 인터페이스가 아니기 때문에 람다식을 사용하기 위해서는 구현객체가 필요하다. Runnable은 펑셔널 인터페이스로 run() 메서드를 한개 가지고 있는데 이 Runnable인터페이스를 implements 받은 새로운 객체를 생성하고 그 객체를 통해서 람다식을 쓸 수 있다.
- ※functional interface : 메서드가 단 한개인 인터페이스.

*쓰레드 생성 2가지 방법 :

```java
[1]
class A extends Thread{
	public void run(){}
}
A o=new A();
o.start();

-----------------------------------------------
[2]
class A extends AA implements Runnable{
	public void run(){}
}
A o=new A(); 
Thread t=new Thread(o); t.start();
```

```java
class ManyClient {
    StringBuffer str = new StringBuffer("ABCDE");

        ManyClient mc = new ManyClient();
        mc.makeThread("가나다라마");
        mc.makeThread("abcde");

        Thread.sleep(3000);
```

```java
    public void makeThread(String s) {
        /*
        Runnable r=new Runnable(){   //Runnable 인터페이스. 인터페이스는 객체를 만들 수는 없지만 내부무명 클래스로서는 선언할 수 있음.

        Thread t = new Thread(r);
```
